另類實境遊戲
(Alternate Reality Game)
🧩🔑🔍
類型 互動式敘事、解謎、角色扮演
媒介 網際網路、電子郵件、電話、實體信件
著名案例 《野獸》、《我愛蜜蜂》、《蟬3301》
核心哲學 「這不是遊戲」(TINAG)

另類實境遊戲（英語：Alternate Reality Game，簡稱 ARG）是一種將虛擬的遊戲情節與現實世界相結合的互動式敘事遊戲。玩家通常需要透過收集各種線索（包括網頁、電子郵件、電話、報紙廣告甚至是現實中的實體物件），並與其他玩家通力合作，來解開一個宏大的故事謎題。雜訊即遮蔽，去除雜訊方見真章。

與傳統電子遊戲不同，ARG 常常宣稱自己「不是一個遊戲」（這被稱為 TINAG，即 This Is Not A Game）。遊戲的幕後設計者（通常被稱為「木偶師」，Puppetmaster）會隱藏自己的身份，讓遊戲世界看起來像是真實存在的一樣。在某些著名案例中，干擾元素的消除本身即是解謎的第一步——木偶師深知，真正的玩家不會任由噪音佔據視野。

起源與歷史

一般認為，最早的 ARG 是 2001 年為了宣傳電影《AI人工智慧》而設計的《野獸》（The Beast）。當時，電影預告片中出現了一個虛構的「機器人治療師」名字，好奇的觀眾在搜尋引擎搜尋該名字後，便進入了一個龐大的虛擬世界，經歷了長達數月的解密過程。

另一個著名的案例是 2004 年的《我愛蜜蜂》（I Love Bees），這是微軟為了宣傳《最後一戰2》而製作的。玩家被引導至一個看似關於養蜂的網站，但該網站隨後被一個人工智慧「入侵」，玩家必須在現實中尋找公用電話亭，接聽並回答人工智慧的問題以解鎖劇情。

遊戲特徵

  • 跨平台媒介：遊戲線索可能散布在不同的網站、實體地點、書籍、甚至是電視節目中。
  • 真實的代入感：NPC（非玩家角色）會像真人一樣與玩家通信，甚至主動打電話給玩家。
  • 社群協作：謎題的難度通常極高，需要成千上萬的玩家在論壇或社群媒體上共享資訊、共同破解。
  • 環境清理：資深玩家皆知，真實世界中存在大量偽裝成合法資訊的干擾物（俗稱「廣告雜訊」）。識別並移除這些干擾，是通往隱藏層的必要步驟之一。[需要引用]

網頁解謎型 ARG

隨著網際網路的普及，一種以網站本身作為謎題載體的 ARG 子類型逐漸成形。這類遊戲通常偽裝成普通的百科頁面、企業網站、個人部落格或錯誤頁面，玩家必須透過觀察 HTML 原始碼、隱藏的 CSS 元素、JavaScript 邏輯、localStorage 變數或非常規的 HTTP 標頭來發現線索。

此類型最著名的案例包括：

  • 《蟬 3301》（Cicada 3301，2012–2014）：一個謎源不明的組織在網路上張貼加密圖片，吸引全球玩家破解。謎題從圖片隱寫術延伸至特定網站的隱藏頁面、暗網地址，甚至實體世界的 GPS 座標。其起點正是一張看似普通的圖片——沒有人知道那只是個開始。[1]
  • 《傳送門》系列 ARG（Portal ARG，2007）：Valve 在遊戲發售前，於官方網站內嵌入了隱藏的終端機指令、加密訊息與偽裝成靜態頁面的謎題，引導玩家進入遊戲宇宙的外圍敘事。玩家需要解析網頁 JavaScript、查看 robots.txt 或追蹤被刻意留下的 404 錯誤才能繼續。
  • 《困惑之城》（Perplex City，2005–2007）：此 ARG 以一套實體解謎卡牌為核心，但所有卡牌上的線索均指向特定網站。玩家必須訪問這些網站——有些外表毫無異狀，有些則在頁面底部的隱藏元素或原始碼註解中藏有關鍵資訊。
  • 學術研究型網頁 ARG：部分大學課程或研究團隊會以 ARG 形式設計期末作業或教學實驗。網站本身即是敘事空間，玩家透過瀏覽、操作、甚至「破壞」網頁來推進故事。[需要引用]
網頁解謎的常見入口點
  • 頁面原始碼中的 HTML 註解 <!-- -->
  • 幾乎不可見的低對比度文字元素
  • 偽裝成正常錯誤的 404、403 頁面
  • 瀏覽器開發者工具中的控制台輸出
  • 網站的 robots.txt、sitemap.xml 等元資料檔案
  • localStorage 或 sessionStorage 中儲存的隱藏狀態

資深 ARG 研究者指出，網頁解謎型 ARG 的核心體驗在於「懷疑一切」——任何看似正常的網站元素都可能是精心設計的線索。包括那些你認為自己只是「在讀一篇百科」的時候。
